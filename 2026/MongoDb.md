# 
    介于关系型数据库和非关系数据库之间
    1、节省内存，实现冷热数据（热数据放内存，冷数据放硬盘）
    2、它是一个高并发情况下快速写入海量数据的产品
    3、它可以写筛选条件

# mongodb的几个属性
    1、database 数据库
    2、collection 集合
    3、docment 文档

# 和关系型数据库比较
    1、不需要提前设计表结构，拓展性高
    2、写数据用json格式
    3、MySQL能做的事mongodb大都能做，但mongodb不能多表联查

# 使用场景
    1、分布式下面的日志记录
    2、传感器记录
    3、滴滴打车、外卖  地理位置
    4、数据爬虫
    5、海量数据的统计（聚合函数），redis做不到
