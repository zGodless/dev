# 介于关系型数据库和非关系数据库之间
    1、节省内存，实现冷热数据（热数据放内存，冷数据放硬盘）
    2、它是一个高并发情况下快速写入海量数据的产品
    3、它可以写筛选条件

# mongodb的几个属性
    1、database 数据库
    2、collection 集合
    3、docment 文档

# 和关系型数据库比较
    1、不需要提前设计表结构，拓展性高
    2、写数据用json格式
    3、MySQL能做的事mongodb大都能做，但mongodb不能多表联查

# 使用场景
    1、分布式下面的日志记录
    2、传感器记录
    3、滴滴打车、外卖  地理位置
    4、数据爬虫
    5、海量数据的统计（聚合函数），redis做不到

# 集群

 ##　第一种：一个主，多个从
    一个主挂了，从库随机选一个变成主库，等原来的主库好了，会变成从库
 ## 第二种：一个主，多个从，一个仲裁节点（不储存数据）
    一个主挂了，由仲裁节点决定哪个从库变成主库

# 索引
    mongodb只有两种索引
    一种是普通索引：单个索引、复合索引等
    一种是唯一索引：字段的值必须唯一

    只有经常会发生改变的字段要加索引

# 事务
    mongodb支持事务，但要4.0版本以上并且必须要集群
